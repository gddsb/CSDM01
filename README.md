# 长沙大满生产制造系统 (Milk Can MES)

基于 React + Ant Design + Vite + Node.js + SQLite 构建的奶粉罐生产制造执行系统，为东莞市大满包装实业有限公司长沙分公司提供生产管理、质量管理、设备管理、数据大屏及报表中心等核心业务模块。

## 功能模块

### 系统管理
- 用户管理：用户增删改查、角色分配、启用/禁用
- 角色权限：角色定义与权限管理
- 菜单管理：菜单/权限的树形结构管理
- 系统配置：多页签配置（参数配置、项目环境、数据库配置、备份还原），系统版本只读
- 操作日志：用户操作行为审计

### 基础数据
- 料品档案：料品基础信息（分类、规格、尺寸、重量、生效日期等），支持关联客户
- 客户档案：客户基础信息（分类、信用等级、生效失效日期、税务银行信息等）
- 产线档案：生产线配置与状态管理
- 工序档案：生产工序定义与顺序调整
- 制程不良分类：不良类型与单位定义，支持树形多级分类
- 编码管理：系统自动编号规则配置，支持自由组合前缀、日期格式、分隔符、序号位数、重置周期

### 生产管理
- 生产订单：订单创建、下发、关闭全生命周期，订单号自动生成（MO-16+YYMMDD+3位序号），完工后查看关联工单/人员/异常记录
- 工单列表：工单全生命周期管理（开立→开工→关闭→完工），自动计算工时
- 生产报工：工序级报工，投入/产出/不良登记
- 人员记录：工单人员配置与班次管理，工单开工自动创建
- 异常记录：生产异常登记，关联订单与工单

#### 生产订单业务流程
- **状态机**：`开立 → 已下达 → 已关闭`
- **订单号规则**：`MO-16` + `YYMMDD` + 3位流水号（如 `MO-16260704001`）
- **新增订单**：料品下拉仅显示 C 开头的成品罐料品，选中料品后自动带入料号、料品名称、规格、菲林编号、版本
- **筛选查询**：支持按订单号、料号、状态、计划时间区间查询；输入框失焦/状态下拉变更/日期选择完成时立即触发筛选，并提供查询按钮与重置功能
- **编辑约束**：仅"开立"状态可编辑，可修改数量、计划开始日期、计划完成日期；计划开始日期不得早于今天，计划完成日期不得早于计划开始日期
- **操作权限**：开立状态下可"下发/编辑/删除"，已下达状态下可"查看/关闭"，已关闭状态下仅可"查看"

### 质量管理
- 检验标准管理：检验标准定义与版本控制
- 来料检验 / 过程检验 / 成品检验：全流程质量检验
- 产品微生物检验 / 环境检验
- 投诉管理：客诉管理与供应商投诉
- 检测仪器管理：仪器校准记录

### 设备管理
- 设备档案：设备基础信息与特种设备检定
- 点检记录 / 维修保养 / 设备OEE

### 数据大屏
- 生产实时看板：工单进度与产线状态
- 质量分析看板
- 管理驾驶舱：综合数据展示

### 报表中心
- 生产日报 / 质量月报 / 效率分析
- 生产报表 / 质量报表 / 异常分析报表

## 快速开始

### 前端

```bash
# 安装依赖
npm install

# 启动开发服务器
npm run dev

# 构建生产版本
npm run build

# 预览生产构建
npm run preview
```

前端开发服务器默认运行在 `http://localhost:5173`

### 后端

```bash
# 进入后端目录
cd server

# 安装依赖
npm install

# 初始化数据库（创建表结构和默认数据）
node src/seed.js

# 启动后端服务
node src/app.js
```

后端服务默认运行在 `http://localhost:3001`

### 数据库操作

```bash
# 清空所有业务数据（保留角色和 admin 用户）
cd server
node src/clean-init.js

# 重新初始化完整数据
cd server
node src/seed.js

# 导入料品档案种子数据（114 条料品记录）
cd server
node src/import-materials.js
```

## 料品档案表（bas_material）

料品档案表共 24 个字段，涵盖料品基础信息、尺寸规格、印刷工艺、生效失效等属性：

| 分类 | 字段示例 |
|------|---------|
| 标识 | material_id、material_code、material_name、specification、version_no、film_no |
| 分类 | material_category、material_type、unit、status |
| 尺寸 | can_diameter、can_height、can_thickness、cover_diameter、cover_thickness |
| 重量 | net_weight、gross_weight、tare_weight |
| 工艺 | printing_process、color_count、coating_type |
| 日期 | effective_date、expiry_date |
| 审计 | created_at、updated_at |

种子数据通过 `server/src/import-materials.js` 导入，包含 114 条料品记录（含 27 条 C 开头的成品罐料品）。

## 功能特性

- 多主题切换（纯牛奶色、深色工厂等）
- 响应式布局，支持侧边栏折叠
- 表格表头居中、操作列统一宽度
- 右侧抽屉（Drawer）查看详情
- 工时按 0.5 小时取值计算
- 工单完工后关联记录自动转为只读
- 生产订单详情展示关联工单、人员、异常记录
- 生产订单状态机统一为"开立 / 已下达 / 已关闭"，订单号自动生成（MO-16+YYMMDD+3位流水号）
- 生产订单筛选支持失焦即查、状态变更即查、日期选择即查
- 生产订单编辑模式与新增模式日期约束一致（计划开始 ≥ 今天，计划完成 ≥ 计划开始）
- 料品名称列宽最小 200，内容过长时自动换行
- 料品档案支持完整的尺寸、重量、印刷工艺等属性（24 字段）
- JWT 认证，自动处理 token 过期重定向
- 系统配置多页签页面：参数配置、项目环境、数据库配置、备份还原
- 编码管理：启用即审核，同一表单字段仅一个生效规则，启用新规则自动停用旧规则
- 编码规则操作列仅显示可操作按钮，已审核规则隐藏编辑和删除按钮
- 编码规则列表表不显示"编码规则"字段；不保留任何测试规则记录

## 大屏看板规则

### 数据展示
- **当天数据优先**：所有大屏（生产/管理/质量）默认显示当天数据；当天无数据时自动回退到上一个有数据的日期
- **工序产出统计 / 工单实时进度**：仅定期更新数据，禁用 ECharts 动画（`animation: false`、`animationDuration: 0`、`animationDurationUpdate: 0`）
- **生产订单概览**：按屏幕可显示的记录数渲染，排序顺序为 `已开工 → 待下达 → 已关闭`
- **产线运行状态 / 设备状态**：隐藏滚动条（使用 `.bs-no-scrollbar` 类，兼容 Firefox / IE / Chrome / Safari）
- **工序产出统计 与 工单实时进度**：卡片宽度平均分配（`flex: 1` 等分）

### 闲置态与环境数据
- **15 秒无操作自动隐藏**：监听 `mousemove` / `mousedown` / `keydown` / `touchstart` / `wheel` 事件，超过 15 秒无操作时自动隐藏左上角看板名称按钮与切换 Tab，仅显示系统时间（HH:MM:SS）+ 在线指示灯；任意操作立即恢复
- **右上角环境数据**：实时显示"温度 / 湿度 / 压差"三项数据
  - 温度范围 20~23°C，湿度范围 58~63%，压差范围 15~21Pa
  - 每 8 秒刷新一次，每次变化与上一次数值差不超过 2
  - 数据保留小数点后一位
  - 数据采用平滑随机算法（超出边界时镜像反射回区间内）

### 顶部主题切换
- **单图标切换**：顶部主题区只显示一个圆形图标按钮，点击后自动切换为下一个主题
- **主题循环顺序**：纯净奶源 → 暗夜工厂 → 蓝天牧场 → 金属质感 → 自然绿洲 → 暖阳琥珀（循环往复）
- Tooltip 提示当前主题名称与"点击切换"

## 用户设置与头像

- **预设头像**：系统内置 20 种预设头像（基于 DiceBear API，涵盖 avataaars / fun-emoji / bottts 三种风格），用户在"用户设置 → 更换头像"中可任选其一
- **自定义上传**：支持上传自定义头像（仅图片格式，单文件最大 2MB）；上传后系统自动删除旧的自定义头像文件
- **存储位置**：自定义头像保存至 `uploads/avatars/`，临时上传目录为 `uploads/tmp/`
- **静态服务**：后端通过 `express.static` 服务 `/uploads` 路径，浏览器缓存 7 天
- **接口**：
  - `POST /api/users/me/avatar`（multipart/form-data，字段名 `avatar`）上传自定义头像
  - `PUT  /api/users/me/avatar`（body: `{ avatar_url }`）设置预设头像
  - `PUT  /api/users/me/profile` 更新姓名/手机/邮箱

## 数据库配置与迁移

- **数据迁移**：系统配置 → 数据库配置 提供"数据迁移"卡片，支持在 SQLite / MySQL / PostgreSQL / SQL Server 等常见数据库环境之间迁移
- **迁移前自动备份**：执行迁移前系统自动对当前 SQLite 数据库进行全量备份（备份文件存放于 `server/data/backups/`），可在"备份还原"页面选择该备份进行还原
- **迁移流程**：测试目标库连接 → 逐表 `findAll` + `bulkCreate` 复制数据 → 更新 `.env` 文件中的 `DB_DIALECT` / `DB_HOST` / `DB_PORT` / `DB_NAME` / `DB_USER` / `DB_PASSWORD` → 返回迁移统计
- **接口**：
  - `GET  /api/config/database/migration-targets` 查询可迁移的目标数据库列表
  - `POST /api/config/database/migrate` 执行数据迁移（需"系统配置"操作权限）

## 项目环境信息

- 系统配置 → 项目环境 页面展示以下系统级信息：
  - **磁盘**：磁盘总容量、已用、可用、已用百分比、挂载点
  - **操作系统**：操作系统版本、类型、内核版本、主机名、系统运行时长
  - **运行时**：Node.js 版本、CPU 架构、内存使用情况
  - **技术栈**：完整展示前端与后端技术栈（各依赖名称、版本、分类），版本号从 `package.json` 实时读取
  - **重启服务**：一键重启后端服务（需"系统配置"操作权限），重启过程自动等待服务恢复并刷新环境信息
